# Leveraging LLMs for Activity Extraction from
Email

This repository provides supplementary material for the paper:

**"Leveraging Large Language Models for Business Process Activity Extraction from Email Communication"**

## 📌 Overview

This repository contains representative examples and experimental artifacts used to support the evaluation of Large Language Models (LLMs) for extracting:

- business process activities  
- associated business data  

from domain-specific email communication.

The objective of the study is to assess how LLMs can capture both explicit and implicit process-related information from unstructured email text.

---

## 📂 Repository Structure

The repository includes the following components:

### 🔹 Prompts
- `prompts/main_prompt.txt`  
  The prompt used for activity and business data extraction.

### 🔹 Representative Examples
- `examples/UC-1/`, `UC-2/`, `UC-3/`  
  Each use case contains:
  - `email.txt`: anonymized email content  
  - `gold.json`: manually annotated ground truth  
  - `llm_gpt4o.json`: output generated by GPT-4o  
  - `rule_based.txt`: output of the frequent-pattern approach  

These examples illustrate typical success and failure behaviors of the evaluated approaches.

---

## 🔁 Reproducibility

This repository supports **qualitative reproducibility** of the experiments.

It provides:
- the prompt used in the experiments  
- representative anonymized examples  
- gold annotations  
- model outputs  

These artifacts allow readers to:
- understand the extraction methodology  
- inspect model behavior  
- compare LLM-based and rule-based outputs  

---

## 📊 Data Availability

The experiments are conducted on the publicly available Enron email dataset:

https://www.cs.cmu.edu/~enron/

Due to the size of the dataset and the preprocessing required to construct the evaluation subset, the dataset is not redistributed in this repository.

Instead, we provide anonymized representative examples and experimental artifacts to ensure transparency and reproducibility of the evaluation.

---

## 🔒 Anonymization

All shared email examples have been manually anonymized to remove:
- personal identifiers  
- sensitive organizational information  

The purpose of the released examples is to illustrate representative extraction behaviors while preserving privacy.

---

## 🎯 Purpose of the Repository

This repository is intended to:
- support transparency of the evaluation  
- illustrate typical extraction outcomes  
- provide insight into model strengths and limitations
